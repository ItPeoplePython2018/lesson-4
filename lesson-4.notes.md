1. Введение в Django
--------------------

- Для чего нужен Django.
- Web framework.
- Плюсы Django.

2. Создание проекта
-------------------

- `virtualenv`
- `django-admin` и `manage.py`

3. Структура созданого проекта
------------------------------

Созданый проект имеет следующую файловую структуру:

- `manage.py`               - консольное приложение позволяющее взаимодействовать с Django.
- `<project>`               - Python'овский пакет с проектом Django
- `<project>/settings.py`   - Настройки Django
- `<project>/urls.py`       - основные пути (URL'ы) для проекта

4. Основные консольные команды
------------------------------

- `python manage.py runserver` 	- запустить тестовый web-сервер. Данная команда преднозначена только для разработки проекта!!!
- `python manage.py migrate` 	- накатить изменения в базу данных.
- `python manage.py shell`	- запустить интерактивный шелл
- `python manage.py dbshell`	- запустить шелл базы данных

5. Настройки прокета Django
---------------------------

- `DEBUG` 			- показывать отладочную информацию при ошибках.
- `LANGUAGE_CODE` 		- язык приложения
- `USE_I18N` и `USE_L10N` 	- для локализация и интернационализация
- `USE_TZ` 			- использовать таймзоны для дат

6. Создание нового приложения
-----------------------------

```bash
django-admin startapp articles
django-admin startapp articles course/articles
```

- Настройка `INSTALLED_APPS` - список приложений доступных Django

```python
INSTALLED_APPS = [
    ...,
    'coures.articles',
]
```

7. Состав приложения
--------------------

- `<app>/admin.py`  - виды для генерации Django-админки
- `<app>/apps.py`   - приложение для регистрации в Django
- `<app>/models.py` - в данном модуле хранятся модели для работы с БД или другими источниками данных.
- `<app>/tests.py`  - тесты
- `<app>/views.py`  - виды
- `<app>/forms.py`  - формы
- `<app>/templates` - шаблоны
- `<app>/static`    - статика
